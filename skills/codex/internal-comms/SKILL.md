# Internal Comms

## Overview

Use this skill to write crisp internal updates that are useful, honest, and easy to scan. It is adapted from useful patterns in Anthropic's Apache-2.0 `internal-comms` skill, with generic formats for future projects.

## Choose The Format

- 3P update: Progress, Plans, Problems.
- Status report: current state, completed work, next steps, blockers, risks.
- Leadership update: decision needed, impact, options, recommendation, timeline.
- Incident report: impact, current status, timeline, root cause if known, actions.
- FAQ: question, short answer, detail, owner/source.
- Launch note: what changed, who it affects, rollout, support path, known limitations.
- Handoff: context, files/systems touched, verification, risks, next owner.

## Writing Rules

- Put the key takeaway first.
- Separate confirmed facts from assumptions.
- Name dates, owners, and next actions.
- Avoid fake certainty, hype, and unverified status claims.
- Keep paragraphs short and skimmable.
- Include blockers and risks plainly.
- Match the audience: tactical for team, decision-focused for leadership.
